Timetable

  • Challenge Registration and Dry run period**: Janurary 16 ~ May 1, 2023
  • Final Submissions: April 30 ~ May 1, 2023 (GMT+0 or UTC) (one-day only)
  • Winner & Runner-ups Code Submission (see guidelines below): May 4, 2023 (GMT+0 or UTC)
  • Winner Announcement: May 25, 2023
** Dry-run phase: A set of dry-run (validation) data is provided for debugging purpose only. This set is provided just to validate your prediction format (so you can work with our final testing in the end). The Top-1 Accuracy on this set is NOT accurate. Feedback are provided on this dry-run (validation) set only.

Prizes

  • We have two sub-tracks for track 2 this year. The two sub-tracks are ranked separately.
  • Please note that due to the sponsership issue, we will not have cash awards for track 2 this year (winner/runnerup certificates will still be presented). Instead, under a partnership with IEEE transactions on Computational Imaging (TCI), all participants are encouraged to submit their work to TCI for an expedite review process. Please see Paper track for details.

Winner Validation and Confirmation Process

Teams with Top-3 on the final testing Codalab leaderboard would be required to submit their code for the winner validation and confirmation process. Please pack all the required items into one single .zip file and sent it (or the link) to cvpr2023.ug2challenge@gmail.com by May 5th. The required items (of both subtrack 2.1 and 2.2) are as follow:

  1. Your code and trained weight of your model that could lead to the result shown in the learderboard. You should also provide a script to run the testing process on the final testing stage data. Please include any necessary instructions and requirements to run the code. We will reproduce your result using the submitted pre-trained model.
  2. A complete description of your method used in this challenge.
  3. The configuration of the simulator used to generate the training data and the description of the ground truth dataset. This can be in the form of a python script, or text descriptions. (If the simulator is used. )
  4. A description of external data used, if any.

Rules

Read carefully the following guidelines before submitting. Methods not complying with the guidelines will be disqualified.

  • We encourage participants to use the provided training and validation data for each task, as well as to make use of their own data or data from other sources for training. However the use any form of annotation or use of any of the provided benchmarks test sets for either supervised or unsupervised training is strictly forbidden.
  • Team name of submissions on Codalab must match the registration information. Any submission with a team name not registered will not be qualified for prizes. Only a single submission per team can be the winner of a single sub-track of this sub-challenge. Changes in algorithm parameters do not constitute a different method, all parameter tuning must be conducted using the dataset provided and any additional data the participants consider appropriate.
